Welcome to Intelligent Predictive Maintenance For An Example Industrial IoT Device Project Website COMP4920 YAŞAR University

















IoT devices provide many opportunities for the maintenance of industrial devices and maintaining efficiency in industrial enterprises. Unexpected machine faults reduce productivity in industrial enterprises and cause a loss of income for the enterprise. In this project, sensor data from an industrial IoT device is processed and required maintenance is determined by a machine learning model that reports different faults. Classified faults are registered with timestamps to a database. In the realization of this project, simulated sensor data is used.

Problem Of the Industry

Non detected machine failures can cost big financial losses to enterprises.

Detecting faults on asynchronous machines can have very wide application area since they are frequently used in different industries.

Detecting machine anomalies in a detailed way helps industrial enterprises to plan accurate maintenance plan.

Solution We Offer

Produce a predictive maintenance algorithm that warns users of anomalies in the machine, with the help of machıne learnıng. Different fault conditions can re recognized on asynchronous three phase electrical motors using this Project. THIS WILL HELP INDUSTRUIAL ENTERPRISES TO SOLVE MACHINE PROBLEMS IN AN EASIER WAY.

Thanks to the predictive maintenance algorithm, necessary maintenance will be PLANNED AS SOON AS A MACHINE ANOMALY IS REPORTED.

Our Project's Goal

To monitor the condition of the machines for better usability and performance, to ensure that they are maintained on time and with attention to the machine's necessary parts, and to optimize them.

How Predictive Maintenance for Industrial IoT Devices Project Works ?

Actors

User

The user can be the person who can control the system movements through the GUI, this person can be the machine administrator or the people who have permission to use the machine.

PM System

The PM System works to make the highest prediction of accuracy and performance for the machine, based on the commands it receives and the models it is trained on.

Detailed Design

The GUI can be defined as a user interface that enables the user to access the system and activates the transactions by transmitting the user's orders to the system.

UML Class Diagram Of The Designed System Of Project

Description Of The Classes

GUI Helper Class

The purpose of GUI_Helper class is to be a bridge between user and the system. GUI_Helper realizes a user interface that provides user necessary information such as fault timestamps and sensor data plots. GUI takes commands from the user and encapsulates workflow defined in other classes.

ML Model Realization Class

The purpose of ML Realization class is to perform machine learning activities. We train our ML Model with numerous SIMULINK simulations for it to have a complete understanding of different fault classes. The best accuracy value and the best performance is satisfied in a feasible training time using a tree model. The functions of all methods and variables are explained in detail below.

Database Helper Class

The purpose of the Database Helper class is to provide access to the databases to be used in the Project. It defines both read and write operations. It also gives other classes the ability to use data. The detailed description of all methods and variables of the class is given below.
